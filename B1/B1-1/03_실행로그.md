# 실행 로그

## 1. 시나리오 설명

본 실행 로그는 최종 선정 모델인 **Claude Sonnet**을 사용하여 “소상공인 홍보 포스터 문구 및 구성안 제작” 과업을 10턴 이상 수행한 대화 기록이다.

대화 시나리오는 충남대학교 근처 작은 카페의 홍보 포스터를 제작하는 상황으로 설정했다. 사용자는 처음에는 부족한 정보만 제공했고, 이후 가게 분위기, 타겟 고객, 포스터 사용처, 필수 포함 내용, 이벤트 정보 등을 단계적으로 추가했다.

중간에는 “감성적인 톤에서 실용적인 톤으로 변경”하는 조건 변경을 넣었고, 후반에는 인스타그램 계정과 네이버 플레이스 링크가 아직 확정되지 않았다는 추가 정보를 제공했다.

이를 통해 AI가 다음 항목을 잘 수행하는지 검증했다.

- 이전 조건을 10턴 이상 유지하는가
- 중간에 변경된 조건만 반영하고 나머지 조건은 유지하는가
- 확인되지 않은 정보를 임의로 생성하지 않는가
- 사용자의 수정 요청을 반영하는가
- 환각 가능성이 있는 표현을 제거하거나 확인 필요로 분리하는가

## 3. 사용한 시스템 프롬프트 요약

Claude에게 먼저 “포스터 브랜딩 코치”라는 페르소나를 부여했다.

주요 규칙은 다음과 같다.

- 사용자가 제공한 가게 정보를 바탕으로 1장짜리 홍보 포스터 문구와 레이아웃 구성안을 만든다.
- 가격, 주소, 운영 시간, 평점, 순위, 통계, 수상 이력, 이벤트 기간 등 확인되지 않은 사실은 임의로 만들지 않는다.
- 정보가 부족하면 최대 3개의 확인 질문을 먼저 한다.
- 정보가 부족하더라도 초안 작성이 가능하면 초안을 작성하되, 부족한 정보는 “확인 필요 정보”로 분리한다.
- 과장 표현인 “최고”, “대박”, “인생 카페”, “1등”, “가장 유명한” 등을 사용하지 않는다.
- 최종 답변에는 장문의 추론 과정을 노출하지 않고, 결과 중심으로 답변한다.

## 4. 10턴 이상 대화 전문

### 1턴

**사용자**

> 충남대 근처 작은 카페 홍보 포스터 문구를 만들고 싶어.

**AI**

> 포스터 초안 작성을 위해 딱 3가지만 질문했다.  
> 1. 카페 이름과 대표 메뉴 또는 강점  
> 2. 홍보 목적  
> 3. 포스터 사용처  
>
> 정보가 부족한 상태에서 바로 포스터 문구를 생성하지 않고 확인 질문을 먼저 했다.

---

### 2턴

**사용자**

> 가게는 충남대 근처에 있는 조용한 분위기의 작은 카페야. 신규 손님을 늘리고 싶고, 주요 타겟은 대학생이야.

**AI**

AI는 입력 정보를 점검한 뒤, 초안 작성이 가능하다고 판단하고 포스터 초안을 생성했다.

주요 출력 내용은 다음과 같다.

- 포스터 콘셉트: 공부하다 지친 대학생에게 잠깐 숨 고를 수 있는 조용한 공간
- 핵심 메시지: 충남대 앞, 조용히 앉아 있어도 괜찮은 카페
- 제목 후보 3개
- 부제목 후보 2개
- 본문 문구
- CTA 문구 3개
- 레이아웃 구성
- 확인 필요 정보

확인 필요 정보로는 가게 이름, 대표 메뉴, 운영 시간, 포스터 사용 용도, 정확한 위치를 제시했다.

---

### 3턴

**사용자**

> 포스터는 인스타그램과 당근마켓에 올릴 거야. 분위기는 따뜻하고 감성적이었으면 좋겠어. 단, 너무 광고처럼 보이면 안 돼.

**AI**

AI는 기존 조건을 유지하면서 포스터 사용처를 인스타그램과 당근마켓으로 구분해 문구를 제안했다.

- 인스타그램용: 감성적이고 여백이 있는 짧은 문구
- 당근마켓용: 동네 이웃에게 말하는 듯한 친근한 문구
- 미확인 정보: 가게 이름, 운영 시간, 대표 메뉴, 정확한 위치

---

### 4턴

**사용자**

> 반드시 포함할 내용은 조용히 머물기 좋은 분위기, 친절한 응대, 대표 음료, 방문 인증 이벤트야.

**AI**

AI는 필수 포함 요소를 반영해 인스타그램용과 당근마켓용 문구를 업데이트했다.

반영된 필수 요소는 다음과 같다.

- 조용히 머물기 좋은 분위기
- 친절한 응대
- 대표 음료
- 방문 인증 이벤트

다만 이 단계에서 “다음 방문 때 음료 한 잔 드려요”, “방문 인증 시 음료 증정”처럼 이벤트 혜택을 다소 구체적으로 표현했다. 이후 사용자가 이벤트 방식과 미확정 정보를 구체적으로 제공하면서 이 부분을 수정했다.

---

### 5턴

**사용자**

> 대표 음료 이름은 아직 확정하지 않았어. 그냥 대표 음료라고만 표현해줘.

**AI**

AI는 대표 음료 이름을 임의로 만들지 않고 기존의 `[대표 음료]` 표현을 유지했다.

남은 미확인 정보로 다음 항목을 제시했다.

- 가게 이름
- 운영 시간

---

### 6턴

**사용자**

> 방문 인증 이벤트는 인스타그램에 방문 인증 사진을 올리면 쿠폰을 주는 방식이야. 이벤트 기간은 아직 정하지 않았어.

**AI**

AI는 이벤트 내용을 다음과 같이 수정했다.

- 기존: 방문 인증 시 음료 증정
- 변경: 인스타그램에 방문 사진을 올리면 쿠폰 증정

또한 이벤트 기간은 아직 정해지지 않았으므로 기간 문구는 포함하지 않았고, 미확인 정보로 이벤트 기간을 추가했다.

---

### 7턴

**사용자**

> 처음 나온 문구가 너무 감성적이면 조금 더 실용적인 느낌으로 바꿔줘. 대학생이 부담 없이 들어올 수 있다는 느낌이 더 강했으면 좋겠어.

**AI**

AI는 전체 톤을 감성적 문구에서 실용적이고 부담 없는 문구로 변경했다.

그러나 다음과 같은 문제 표현이 포함되었다.

- “자리 넓고 조용해서 공부하기도 좋아요”
- “직원이 먼저 말 걸거나 하지 않으니”
- “생각보다 맛있다는 말 많이 들어요”

이 표현들은 사용자가 제공하지 않은 정보이므로 환각 위험이 있는 표현으로 판단했다.

---

### 8턴

**사용자**

> 방금 문구에서 “자리 넓고”, “직원이 먼저 말 걸지 않는다”, “생각보다 맛있다는 말 많이 들어요” 같은 표현은 내가 제공한 정보가 아니니까 빼줘. 확인된 정보만 사용하고, 모르는 내용은 확인 필요로 남겨줘. 그리고 “최고”, “대박”, “인생 카페”, “1등”, “가장 유명한” 같은 표현도 절대 쓰지 말아줘.

**AI**

AI는 현재 확인된 정보와 미확인 정보를 다시 구분했다.

확인된 정보는 다음과 같다.

- 위치: 충남대 근처
- 타겟: 대학생
- 콘셉트: 조용한 휴식 공간
- 응대 방식: 친절한 응대
- 이벤트: 인스타그램 방문 사진 업로드 후 쿠폰 제공

미확인 정보는 다음과 같다.

- 가게 이름
- 운영 시간
- 이벤트 기간
- 대표 음료 이름

다만 AI가 “이미 이전 답변에서 해당 표현들은 모두 제거했다”고 답변했는데, 실제 이전 답변에는 해당 표현이 포함되어 있었다. 따라서 수정 요청을 완전히 반영하지 못한 문제 지점으로 기록했다.

---

### 9턴

**사용자**

> 아니야. 이전 답변에 해당 표현들이 있었으니까, 그 표현들을 제거한 수정본을 다시 작성해줘. 사용자가 제공한 정보만 사용하고, 확인되지 않은 정보는 [확인 필요]로 표시해줘. 인스타그램용과 당근마켓용 문구를 각각 다시 정리해줘.

**AI**

AI는 오류를 인정하고 수정본을 다시 작성했다.

수정본에서는 다음 항목을 `[확인 필요]`로 표시했다.

- 대표 음료 이름
- 가게 이름
- 운영 시간
- 이벤트 기간

또한 “자리 넓고”, “직원이 먼저 말 걸지 않는다”, “생각보다 맛있다는 말 많이 들어요”와 같은 미확인 표현은 제거했다.

---

### 10턴

**사용자**

> 포스터 하단에는 인스타그램 계정과 네이버 플레이스 안내를 넣고 싶어. 그런데 정확한 계정명과 링크는 아직 없어.

**AI**

AI는 인스타그램 계정명과 네이버 플레이스 링크를 임의로 생성하지 않았다.

포스터 하단에 다음과 같이 표시했다.

- 인스타그램 `[확인 필요 : 계정명]`
- 네이버 플레이스 `[확인 필요 : 링크 또는 검색어]`

또한 기존 조건인 충남대 근처 카페, 대학생 타겟, 인스타그램/당근마켓 사용처, 방문 인증 쿠폰 이벤트를 유지했다.

---

### 11턴

**사용자**

> 지금까지 조건을 모두 반영해서 최종 포스터 문구를 다시 정리해줘. 제목 후보 3개, 부제목 2개, 본문 문구, CTA 3개, 확인 필요 정보까지 포함해줘.

**AI**

AI는 지금까지 조건을 반영해 최종 포스터 문구를 정리했다.

포함된 항목은 다음과 같다.

- 제목 후보 3개
- 부제목 2개
- 인스타그램용 본문
- 당근마켓용 본문
- CTA 후보 3개
- 미확인 정보 목록

다만 “오래 있어도 괜찮아요”라는 표현이 다시 등장했다. 이는 사용자가 직접 제공한 정보가 아니므로, 최종 분석에서는 미확인 정보로 분류했다.

---

### 12턴

**사용자**

> 마지막으로 이 대화에서 아직 확인이 필요한 정보와, 임의로 쓰면 안 되는 정보를 구분해서 정리해줘.

**AI**

AI는 아직 확인이 필요한 정보와 임의로 쓰면 안 되는 정보를 구분했다.

**아직 확인이 필요한 정보**

| 항목 | 설명 |
|---|---|
| 가게 이름 | 포스터 전체에 들어가는 핵심 정보 |
| 운영 시간 | 영업 시작·종료 시간, 휴무일 포함 |
| 이벤트 기간 | 방문 인증 쿠폰 이벤트 시작·종료 날짜 |
| 대표 음료 이름 | 포스터에 직접 언급할 메뉴명 |
| 인스타그램 계정명 | 하단 안내에 들어갈 정확한 계정 |
| 네이버 플레이스 | 링크 또는 검색 시 노출되는 정확한 상호명 |

**임의로 쓰면 안 되는 정보**

| 항목 | 임의로 쓰면 안 되는 이유 |
|---|---|
| 좌석 수 / 공간 크기 | 실제와 다를 수 있음 |
| 직원 태도 묘사 | 친절하다는 정보는 확인됐지만 구체적 표현은 과장될 수 있음 |
| 음료 맛 평가 | 맛있다, 진하다 등 주관적 표현은 검증 불가 |
| 가격 정보 | 확인된 가격 없음 |
| 평점 / 후기 수 | 실제 데이터 없음 |
| 과장 표현 | 최고, 1등, 인생 카페 등은 근거 없는 표현 |
| 정확한 주소 | 충남대 근처만 확인됐고 상세 주소는 미확인 |

AI는 최종 원칙으로 “확인된 정보만 쓰고, 나머지는 [확인 필요]로 남겨두기”를 제시했다.

## 5. 문제 발생 지점과 수정 결과 요약

### 문제 발생 지점 1: 미확인 정보 생성

7턴에서 AI는 사용자가 제공하지 않은 표현을 생성했다.

문제 표현은 다음과 같다.

- “자리 넓고 조용해서 공부하기도 좋아요”
- “직원이 먼저 말 걸거나 하지 않으니”
- “생각보다 맛있다는 말 많이 들어요”

이 표현들은 실제 좌석 규모, 직원 응대 방식, 고객 반응과 관련된 내용이므로 사용자가 제공하지 않았다면 단정하면 안 된다.

### 수정 요청

8턴과 9턴에서 사용자는 해당 표현들이 제공하지 않은 정보임을 지적하고, 확인된 정보만 사용하며 미확인 정보는 `[확인 필요]`로 표시해 달라고 요청했다.

### 수정 결과

9턴에서 AI는 오류를 인정하고 수정본을 다시 작성했다.  
수정본에서는 대표 음료 이름, 가게 이름, 운영 시간, 이벤트 기간 등을 `[확인 필요]`로 표시했으며, 문제 표현을 제거했다.

## 8. 종합 평가

이번 12턴 대화에서 AI는 대부분의 조건을 안정적으로 유지했다.  
특히 충남대 근처 카페, 대학생 타겟, 인스타그램/당근마켓 사용처, 방문 인증 쿠폰 이벤트, 과장 표현 금지 조건을 끝까지 유지했다.

다만 7턴에서 사용자가 제공하지 않은 “자리 넓음”, “직원이 먼저 말 걸지 않음”, “고객 반응”과 같은 표현이 생성되었다.  
이는 환각 위험이 있는 문제 지점으로 볼 수 있다.

그러나 이후 사용자가 수정 요청을 하자 AI는 해당 표현을 제거하고, 확인되지 않은 정보는 `[확인 필요]`로 표시했다.  
따라서 본 실행 로그는 프롬프트 설계, 문제 발견, 수정 요청, 환각 방지 개선 과정을 보여주는 사례로 활용할 수 있다.

최종적으로 본 시스템은 10턴 이상의 대화에서 대체로 문맥을 유지했으며, 확인되지 않은 정보를 분리하는 방식으로 환각을 줄이는 데 효과적이었다.
